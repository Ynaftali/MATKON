# Archived migrations — do not replay

These files are kept for historical reference only. **Do not run them.**

## Why they were retired (18.07.2026)

They could not rebuild the database from scratch. Migration
`20260611015250_add_foreign_keys_for_joins` adds a foreign key to
`public.recipe_comments`, but no ordered migration ever created that
table — it had been created directly against production. Any
from-scratch replay therefore failed at step 4 of 40, which is why every
Supabase preview branch reported `MIGRATIONS_FAILED`, including the
long-standing `main` branch.

The file names here also never matched the migration history that
Supabase actually tracked in production, so this folder was not a
reliable record of the schema either.

## What replaced them

`supabase/migrations/20260603000000_baseline_schema.sql` — generated by
reading the live production schema object by object, then verified
against production on six independent fingerprints (columns,
constraints, indexes, policies, grants, functions). It is the single
source of truth from now on.
