# MATKON — סבב UX/UI · גרסה 0.1

מסמך מעקב לסבב שיפור העיצוב, דיוק המסכים והתהליכים באפליקציה הציבורית **matkon.co**.

- **תיחום הסבב:** אפליקציה ציבורית בלבד · מובייל בלבד (375px, mobile-first).
- **שיטת עבודה:** מסך-מסך → צילום חי של המצב הנוכחי → הערות (משתמש + Claude) → דיון והחלטה → **לא מתקנים בשלב זה**. בסוף הסבב: ניתוח + הצעת סדר ביצוע ואז יישום (עם הדמיה לפני כל שינוי עיצובי).
- **מקור אמת:** הקובץ הזה, מנוהל ב-git ליד `CHANGELOG.md`.
- **סטטוס פריט:** 🔵 לדיון · ✅ הוחלט לתקן · ⏸️ נדחה · ❌ נדחה/לא רלוונטי.
- **עדיפות:** P0 (חוסם/חשוב) · P1 · P2 (נחמד שיהיה).

> השורות מסודרות לפי סדר מסע המשתמש.

## מסננת אבטחה ופרטיות (כל פריט נבדק מולה — מקור: MATKON_Engineering_Security_Guidelines.docx + הבריף)
כל החלטת UX/UI חייבת לכבד:
- **פרטיות (ISRAEL ATTENTION):** רק מדינה, אף פעם לא עיר/כתובת. תצוגת משתמש = דגל מדינה + דגל ישראל + שם פרטי. **אין אווטרים.**
- **פרטי כברירת מחדל:** מתכון private by default; שיתוף לקהילה = פעולה מפורשת (toggle מתחיל כבוי).
- **הפרדת תוכן:** קהילה מופרד מפרטי; לא לחשוף מתכונים פרטיים בחיפוש/פיד.
- **בעלות:** עריכה/מחיקה לבעלים בלבד; אכיפה בשרת (לא רק הסתרה ב-UI).
- **חשיפה:** לא להציג IDs פנימיים/מייל ב-UI ללא צורך.
- **מודרציה:** לשמר את שער המודרציה + הודעת השגיאה למשתמש בכל שינוי בהוספה/עריכה.
- **שפה נאותה / דגל פלסטין אסור לחלוטין.**
- **נגישות WCAG** (קונטרסט, יעדי מגע, פוקוס) — מטרה מחזקת של הסבב.

### מפרט הכפתור — "גלוס-לייט" (אחיד לכל הצבעים)
- **רדיוס:** `18px`.
- **מילוי:** שכבת סחת-אור עליונה מעל גרדיאנט הגוון:
  `linear-gradient(180deg, rgba(255,255,255,SHEEN), transparent 46%), linear-gradient(135deg, בהיר, כהה)`
  כאשר `SHEEN` ≈ `.16–.18` לצבעי טקסט-לבן, `.26–.28` לצבעי טקסט-כהה (ירוק/צהוב).
- **צללית + רים:** `box-shadow: 0 9px 22px rgba(בסיס,.46)` (הילה רכה) `, inset 0 0 0 1px rgba(255,255,255,.13–.18)` (רים-אור פנימי דק) `, inset 0 1px 0 rgba(255,255,255,.36–.52)` (קו-אור עליון).
- **לחיצה:** `transform: scale(.98)` (עקבי עם ה-Design System הקיים).
- **בלי** קונטור שטוח, בלי תלת-ממד כבד, בלי ניאון.

_ליישום: להגדיר כ-CSS custom properties + מחלקת `.btn-glossy` עם וריאנטים per-color. לעדכן את טבלת הצבעים/כפתורים ב-Brief בהתאם._

### מסך 2 — Peek (הצצה) — ✅ אושר 6.7.26 (branch `ux-redesign-preview`, `src/pages/Peek.jsx`)
- הוסרה כותרת הקטגוריה מעל שם המנה (וגם בעתיד). שם המנה: משקל `700`. הוסר אייקון המנעול.

- **4 שקופיות per-אספקט:** (1) כרטיס מתכון "גלו מתכונים..."; (2) 3 אייקוני העלאה צילום=סגול/טקסט=כחול/לינק=צהוב "העלו מתכונים..."; (3) רשימת קניות מקובצת לקטגוריות (עברית+אנגלית+נדיר+איפה להשיג) "ייצרו רשימת קניות..."; (4) מצב בישול — 5 שלבים (2 ✓, פעיל עם טיימר 10:00 + זמן בטקסט, 2 הבאים) "בשלו את המתכון...".

- **"הצטרפות לקהילה"** = כחול-גלוס (~70%), קבוע בתחתית בכל השקופיות כולל האחרונה.

**פתוח להמשך:** שקופיות 3+4 (קניות/בישול) יסונכרנו כשנעצב את המסכים האמיתיים (Shopping / CookingMode). קופי: אופציה לקצר טקסט שקופית 3+4 (לא קריטי).

**גלובלי (פתוח):** תצוגת משתמש לא אחידה בעמודים האמיתיים — Feed (`🇮🇱`+דגל מדינה+שם מלא), RecipePage (ישראל ראשון+שם פרטי), Recipes (בלי דגל ישראל). **לאחד את כולם לדפוס הקנוני** (דגל מדינה→ישראל, שם פרטי+אות ראשונה, בלי עיר).

(מייל חי רק ב-`auth.users`; זו זרימת ה-Confirm-email הרגילה — **לינק, לא קוד**). **החלפת סיסמה** — סיסמה נוכחית + חדשה (עם חוקי-חוזק חיים), כפתור "עדכון סיסמה" → **re-auth** (`signInWithPassword` על הנוכחית) ואז `updateUser({password})` (מונע חטיפת-סשן, [[feedback_highest_security]]); + לינק **"שכחתם את הסיסמה הנוכחית?"** (`forgotPassword`) — **כרגע toast "בקרוב"**, התשתית ב-[[project_account_infra_tasks]].
- **פרטיות ונתונים** (כותרת עם `IconLock` **סגול `#a78bff`** — הקונבנציה מ-Add/EditRecipe): passkeys (כשנתמך); **ייצוא נתונים — כפתור מושבת + "· בקרוב"** (התיאור: "יישלחו ל-Email שלכם"; קוד `exportData` עם fallback Web Share נשאר בקוד; תשתית שליחת-מייל ב-[[project_account_infra_tasks]]); **מחיקת חשבון = כפתור אדום-גלוס מעוצב** (`btn-glossy btn-glossy-red`, טקסט לבן — לא outline; מלא כשמקלידים "מחיקה", גדור מאחורי אישור-הקלדה).
- **יציאה מהחשבון** — כפתור **outline שקט** (`btn-ghost`, החלטת המשתמש: יציאה היא פעולה שגרתית, לא יעד — לא גלוס בולט). נמצא בתחתית; `paddingBottom: calc(var(--nav-h)+24px)` על ה-drawer-body כדי שלא ייחתך מאחורי ה-BottomNav (שניהם `z-index:100`).

**⭐ קומפוננטים משותפים שנוצרו בסשן זה (חוב קומפוננטים):** `src/components/NameFields.jsx` (שם פרטי/משפחה + רמז שם-לטיני), `src/components/PasswordInput.jsx` (שדה סיסמה עם עין + חוקי-חוזק אופציונליים), `src/lib/passwordRules.js` (`PASSWORD_RULES`+`passwordValid`, מקור-אמת יחיד). **`Register.jsx` רופקטר להשתמש בשלושתם** (מרונדר זהה, אומת) — כך שדות הרישום האישיים משותפים כעת בין רישום לעריכת-פרופיל (נקודה 5 של המשתמש). `CountrySelect` כבר היה משותף.

**⚠️ משימות תשתית שנפתחו (מעבר ל-UX, נשמרו ב-[[project_account_infra_tasks]]):** (1) **ייצוא-נתונים-למייל** — דורש ספק מייל (Resend) + API key + הרחבת `api/account.js`; אז להפעיל את הכפתור. (3) **לאמת ב-Supabase שה-"Secure email change" דלוק** (אישור כפול לכתובת ישנה+חדשה).
